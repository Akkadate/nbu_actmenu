# แม่แบบรายการตรวจสอบ UAT (Phase 2)

สภาพแวดล้อม: DEV / UAT / PROD  
วันที่ทดสอบ:  
ผู้รับผิดชอบการทดสอบ:  

## A) ตรวจสอบก่อนเริ่ม
- [ ] ตั้งค่า `.env` ถูกต้อง
- [ ] รัน DB migrations ครบแล้ว
- [ ] Build สำเร็จ
- [ ] Deploy โค้ดล่าสุดขึ้นสภาพแวดล้อมเป้าหมายแล้ว

## B) ทดสอบถดถอยหลัก (ต้องผ่าน)

### B1. การเข้าถึงระบบ Admin
- [ ] เปิด `/admin` แล้วถูกพาไป `/admin/login` เมื่อยังไม่ยืนยันตัวตน
- [ ] ล็อกอินด้วย passkey ได้สำเร็จ
- [ ] ล็อกเอาต์แล้วล้าง session และกลับหน้า login

### B2. การจัดการกิจกรรม
- [ ] สร้างกิจกรรมสำเร็จ
- [ ] แก้ไขกิจกรรมสำเร็จ
- [ ] ปิดใช้งานกิจกรรมสำเร็จ
- [ ] รายการกิจกรรมแสดงฟิลด์ครบตามคาด

### B3. LIFF Core Flow
- [ ] เปิด `/liff?activity=<key>` ด้วย key ที่ถูกต้อง
- [ ] LIFF login ทำงานได้
- [ ] ผู้ใช้ที่ยังไม่ยืนยันตัวตนสามารถ verify สำเร็จ
- [ ] ผู้ใช้ที่ยืนยันแล้วสามารถเช็กอินกิจกรรมได้
- [ ] พฤติกรรมหลังสำเร็จตรงตามที่กำหนด (ใน LINE vs เบราว์เซอร์ภายนอก)

### B4. ความถูกต้องของข้อมูล
- [ ] มีการอัปเดต `line_student_links` เมื่อยืนยันตัวตนสำเร็จ
- [ ] มีการบันทึก `activity_checkins` เมื่อเช็กอินสำเร็จ
- [ ] ป้องกันเช็กอินซ้ำตามกฎ unique ได้

## C) การทดสอบเฉพาะฟีเจอร์ (ขอบเขต Phase 2)
- [ ] Dashboard โหลดและแสดงตัวชี้วัดครบ
- [ ] ตัวกรองข้อมูลแสดงผลถูกต้อง
- [ ] Export CSV ได้ข้อมูลถูกต้อง
- [ ] กฎ Early-warning ให้ผลลัพธ์ถูกต้อง
- [ ] Trigger แจ้งเตือน/ติดตามผลทำงานถูกต้อง

## D) ด้านไม่ใช่ฟังก์ชัน (Non-Functional)
- [ ] ไม่มี console/server error ระดับวิกฤต
- [ ] UI บนมือถือไม่พังในระดับรุนแรง
- [ ] เวลาในการตอบสนองของหน้า/ API สำคัญอยู่ในระดับยอมรับได้
